Nmap (Network Mapper)

The ultimate network discovery and security auditing tool

Overview

Nmap ("Network Mapper") is a free and open-source utility for network discovery and security auditing. It was created by Gordon Lyon (also known by his pseudonym Fyodor Vaskovich) and is used to discover hosts and services on a computer network by sending packets and analyzing the responses.

Key Information

  • Developer: Gordon Lyon (Fyodor)
  • License: GPL-2.0
  • Platform: Cross-platform (Linux, Windows, macOS)
  • Language: C, C++, Python, Lua
  • First Release: September 1997

Features

  • Host Discovery: Identify live hosts on a network
  • Port Scanning: Determine which ports are open on target systems
  • Service Detection: Identify services running on open ports
  • OS Detection: Determine the operating system of target hosts
  • Scriptable Interaction: NSE (Nmap Scripting Engine) for advanced tasks
  • Stealth Scanning: Various scan techniques to avoid detection
  • IPv6 Support: Full IPv6 scanning capabilities
  • Timing Controls: Adjust scan speed and timing

Common Usage Examples

Basic Host Discovery

# Ping scan to discover live hosts
nmap -sn 192.168.1.0/24

# Scan specific IP range
nmap -sn 192.168.1.1-20

Port Scanning

# TCP SYN scan (default)
nmap -sS target.com

# TCP connect scan
nmap -sT target.com

# UDP scan
nmap -sU target.com

# Scan specific ports
nmap -p 22,80,443 target.com

# Scan port range
nmap -p 1-1000 target.com

# Scan all ports
nmap -p- target.com

Service and OS Detection

# Service version detection
nmap -sV target.com

# OS detection
nmap -O target.com

# Aggressive scan (OS, version, scripts, traceroute)
nmap -A target.com

# Script scan
nmap -sC target.com

Stealth and Evasion Techniques

# FIN scan (stealthy)
nmap -sF target.com

# NULL scan
nmap -sN target.com

# Xmas scan
nmap -sX target.com

# Decoy scan
nmap -D RND:10 target.com

# Fragment packets
nmap -f target.com

NSE (Nmap Scripting Engine)

# Run vulnerability scripts
nmap --script vuln target.com

# Run specific script
nmap --script ssh-brute target.com

# List available scripts
nmap --script-help all

# Update script database
nmap --script-updatedb

Timing and Performance

# Timing templates (0-5, 0=paranoid, 5=insane)
nmap -T4 target.com

# Custom timing
nmap --min-rate 1000 --max-rate 5000 target.com

# Parallel scanning
nmap --min-parallelism 100 target.com

Advanced Features

Output Formats

# Normal output
nmap -oN output.txt target.com

# XML output
nmap -oX output.xml target.com

# Grepable output
nmap -oG output.gnmap target.com

# All formats
nmap -oA output target.com

Firewall Evasion

# MTU discovery
nmap --mtu 24 target.com

# Source port specification
nmap --source-port 53 target.com

# Spoof MAC address
nmap --spoof-mac Apple target.com

# Use proxy
nmap --proxies socks4://proxy:1080 target.com

⚠️ Legal and Ethical Considerations

Always ensure you have explicit permission before scanning networks or systems you don't own. Unauthorized network scanning may be illegal in your jurisdiction and could be considered a hostile act. Use Nmap responsibly and only on systems you own or have written permission to test.

Popular NSE Scripts

  • http-enum: Enumerates directories used by popular web applications
  • smb-vuln-*: Tests for various SMB vulnerabilities
  • ssl-enum-ciphers: Enumerates SSL/TLS cipher suites
  • dns-brute: Attempts to enumerate DNS hostnames
  • ftp-anon: Checks if FTP server allows anonymous logins
  • ssh-hostkey: Shows SSH host keys
  • http-title: Shows title of web pages

Best Practices

  1. Start with host discovery before detailed port scanning
  2. Use appropriate timing - faster isn't always better
  3. Combine multiple scan types for comprehensive results
  4. Save results in multiple formats for analysis
  5. Use NSE scripts for deeper enumeration
  6. Document your methodology for reporting
  7. Respect rate limits to avoid detection/blocking
  8. Verify results with manual testing when possible

Integration with Other Tools

Nmap integrates well with numerous other security tools:

  • Metasploit: Import Nmap results for exploitation
  • Nessus: Use Nmap for host discovery
  • Masscan: Combine with Nmap for large-scale scanning
  • Zenmap: GUI frontend for Nmap
  • Custom scripts: Parse XML output for automation
