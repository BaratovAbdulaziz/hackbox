Wireshark

The world's foremost network protocol analyzer

Overview

Wireshark is a free and open-source packet analyzer used for network troubleshooting, analysis, software and communications protocol development, and education. It allows users to see what's happening on their network at a microscopic level and is the de facto standard across many commercial and non-profit enterprises, government agencies, and educational institutions.

Key Information

  • Developer: Wireshark Foundation
  • License: GPL-2.0
  • Platform: Cross-platform (Linux, Windows, macOS)
  • Language: C, C++
  • First Release: 1998 (originally as Ethereal)

Key Features

  • Live Capture: Real-time packet capture and analysis
  • Deep Inspection: Detailed protocol analysis for 3000+ protocols
  • Cross-Platform: Runs on Windows, Linux, macOS, and more
  • Rich VoIP Analysis: Voice over IP call analysis
  • Export Capabilities: Save data in multiple formats
  • Coloring Rules: Visual packet classification
  • Advanced Filtering: Powerful display and capture filters
  • Statistics: Comprehensive network statistics and graphs

Interface Components

Packet List Pane

Shows all captured packets in a list format with basic information like time, source, destination, protocol, and summary.

Packet Details Pane

Displays the protocol tree of the selected packet, showing detailed information about each protocol layer.

Packet Bytes Pane

Shows the raw hexadecimal and ASCII dump of the packet data.

Capture Filters

Applied during packet capture to limit what packets are captured.

# Capture only HTTP traffic
port 80

# Capture traffic for specific host
host 192.168.1.100

# Capture traffic between two hosts
host 192.168.1.100 and host 192.168.1.200

# Capture only TCP traffic
tcp

# Capture traffic on specific network
net 192.168.1.0/24

# Capture traffic not from specific host
not host 192.168.1.1

# Capture specific port range
portrange 1000-2000

Display Filters

Applied after capture to filter the displayed packets.

# HTTP requests only
http.request

# Specific IP address
ip.addr == 192.168.1.100

# Source IP filter
ip.src == 192.168.1.100

# Destination IP filter
ip.dst == 192.168.1.100

# Protocol filters
tcp
udp
dns
arp

# Port filters
tcp.port == 80
udp.port == 53

# Combination filters
http and ip.addr == 192.168.1.100
tcp.port == 80 or tcp.port == 443

# Pattern matching
http.request.uri contains "login"
http.user_agent contains "Mozilla"

Common Analysis Tasks

HTTP Analysis

# Follow HTTP stream
Right-click packet > Follow > HTTP Stream

# Export HTTP objects
File > Export Objects > HTTP

# View HTTP statistics
Statistics > HTTP > Requests

# Filter for HTTP errors
http.response.code >= 400

DNS Analysis

# DNS queries
dns.flags.response == 0

# DNS responses
dns.flags.response == 1

# DNS query for specific domain
dns.qry.name contains "example.com"

# DNS statistics
Statistics > DNS

TCP Analysis

# TCP connection issues
tcp.analysis.flags

# TCP retransmissions
tcp.analysis.retransmission

# TCP reset packets
tcp.flags.reset == 1

# TCP SYN packets
tcp.flags.syn == 1 and tcp.flags.ack == 0

# Follow TCP stream
Right-click packet > Follow > TCP Stream

Security Analysis

Detecting Suspicious Activity

# Large packet sizes (potential data exfiltration)
frame.len > 1500

# Unusual protocols
not (tcp or udp or icmp or arp)

# High frequency connections
Statistics > Conversations > TCP

# Password transmission (unencrypted)
http.request.method == "POST" and http contains "password"

Wireless Security Analysis

# 802.11 management frames
wlan.fc.type == 0

# 802.11 beacon frames
wlan.fc.subtype == 8

# 802.11 deauth frames
wlan.fc.subtype == 12

# WPA handshake
eapol and wlan.addr contains "target_mac"

Command Line Tools

TShark (Terminal Wireshark)

# Basic capture
tshark -i eth0

# Capture to file
tshark -i eth0 -w capture.pcap

# Read from file with filter
tshark -r capture.pcap -Y "http"

# Display specific fields
tshark -r capture.pcap -T fields -e ip.src -e ip.dst -e http.request.uri

# JSON output
tshark -r capture.pcap -T json

Dumpcap

# List interfaces
dumpcap -D

# Basic capture
dumpcap -i eth0 -w capture.pcap

# Ring buffer capture
dumpcap -i eth0 -w capture.pcap -b files:5 -b filesize:100000

⚠️ Legal and Ethical Considerations

Packet capture can reveal sensitive information including passwords, personal data, and confidential communications. Only capture network traffic on networks you own or have explicit permission to monitor. Be aware of privacy laws and regulations in your jurisdiction when using packet analysis tools.

Best Practices

  1. Use appropriate filters to focus on relevant traffic
  2. Save captures regularly to avoid data loss
  3. Use coloring rules to visually identify different traffic types
  4. Enable name resolution carefully as it can slow analysis
  5. Learn keyboard shortcuts for efficient navigation
  6. Use ring buffer captures for long-term monitoring
  7. Validate findings with multiple analysis methods
  8. Document analysis methodology for reports
